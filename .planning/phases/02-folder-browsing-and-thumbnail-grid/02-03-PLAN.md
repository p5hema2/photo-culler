2. Create `usePhotoStore.ts` -- central state management hook (use React useState/useReducer, NOT an external state library):
   - State shape:
     ```typescript
     interface PhotoState {
       folderPath: string | null;
       images: ImageFileInfo[];         // raw scanned images
       classifications: Record<string, Classification>;  // keyed by filename
       sortField: SortField;
       sortDirection: SortDirection;
       filterExtensions: Set<string>;   // active file type filters (empty = show all)
       filterClassification: Classification | null;  // null = show all
       searchQuery: string;
       thumbnailSize: 'small' | 'medium' | 'large';
       groupingThresholdMs: number;
       isLoading: boolean;
       exifProgress: { completed: number; total: number };
       focusedImageId: string | null;   // path of currently focused thumbnail
     }
     ```
   - Actions/methods:
     - `openFolder(folderPath: string)`: set loading, call `window.api.scanFolder`, store images, load results file (restore classifications), trigger EXIF extraction, trigger thumbnail generation for visible images, set `lastFolderPath` in session. Clear previous state first (clear thumbnails via worker.clearAll).
     - `updateExifData(path: string, dateTaken, width, height)`: update the image in state with EXIF data. After all EXIF done, trigger regrouping.
     - `setClassification(filename: string, classification: Classification)`: update classifications map, mark as userOverride=true, trigger debounced save.
     - `cycleClassification(filename: string)`: red -> yellow -> green -> red cycle.
     - `setSortField/setSortDirection/setFilterExtensions/setFilterClassification/setSearchQuery/setThumbnailSize/setGroupingThresholdMs`: standard setters, persist size and threshold to session.
     - `setFocusedImage(path: string | null)`: track keyboard focus.
   - Derived state (useMemo):
     - `filteredImages`: apply extension filter + classification filter + search query
     - `sortedImages`: apply sortImages to filteredImages
     - `groups`: apply groupByTimestamp to sortedImages (import from image-utils)
   - On mount: load session, if `lastFolderPath` exists and folder is accessible, auto-open it.

3. Create `useGrouping.ts` -- thin wrapper that calls `groupByTimestamp` from image-utils and computes group heights:
   - Input: sorted images, thresholdMs, cellSize, containerWidth
   - Output: `PhotoGroup[]` plus a `getGroupHeight(index: number): number` function
   - Height formula: `headerHeight(32) + Math.ceil(group.images.length / imagesPerRow) * cellSize + dividerHeight(16)`
   - `imagesPerRow = Math.floor(containerWidth / cellSize)` (minimum 1)

4. Create `PhotoGrid.tsx`:
   - Uses `@tanstack/react-virtual` `useVirtualizer` with group-level virtualization
   - `parentRef` div: `h-full overflow-auto` (Tailwind)
   - `count: groups.length`, `estimateSize: getGroupHeight`, `overscan: 3`
   - Render virtual items as absolutely positioned divs with `translateY(start)`
   - Each virtual item renders a `<GroupRow>`
   - Track container width via ResizeObserver or `useRef` + layout effect to recompute group heights on resize
   - Call `thumbnailWorker.updateVisibleRange(firstVisibleGroupIndex, lastVisibleGroupIndex)` on scroll
   - THUMBNAIL SIZE MAP: small=120, medium=200, large=300

5. Create `GroupRow.tsx`:
   - Receives: `group: PhotoGroup`, `cellSize: number`, `classifications: Record<string, Classification>`
   - Renders:
     - Group header: lightweight text showing "Series: {count} photos * {timeRange}" and classification summary ("2 keep * 1 review * 5 delete")
     - Time range formatted as "HH:MM:SS -- HH:MM:SS" if startTime and endTime differ, or just "HH:MM:SS" if single
     - Flex-wrap container of ThumbnailCell components
     - HR-style divider at bottom (border-t border-gray-700)
   - Group header styling: `text-xs text-gray-400 px-2 py-1` -- lightweight, not dominating

6. Create `ThumbnailCell.tsx`:
   - Receives: `image: ImageFileInfo`, `cellSize: number`, `classification: Classification`, `isFocused: boolean`, `onClick: () => void`
   - Uses canvas ref to draw ImageBitmap from thumbnail cache (via getThumbnail from context/prop)
   - If thumbnail is 'loading': show gray placeholder with subtle pulse animation
   - If thumbnail is 'error': show broken-image icon placeholder
   - Classification border: 3px solid border around the cell
     - green (keep): `border-green-500`
     - yellow (review): `border-yellow-500`
     - red (delete): `border-red-500`
   - Focus ring: `ring-2 ring-blue-400` when `isFocused`
   - Square cell: `width: cellSize, height: cellSize`, `object-fit: cover` via canvas crop (already done by worker)
   - onClick handler for classification cycling

7. Create `EmptyState.tsx`:
   - Folder icon (SVG or emoji-free text icon) + "No images found in this folder"
   - Centered in grid area
   - Subtle gray styling matching dark theme

All components use Tailwind CSS with dark theme (bg-gray-900 text-white as base from App.tsx).

   - Compact horizontal bar at top of the app (below title bar area)
   - Left section:
     - "Open Folder" button (calls selectFolder)
     - Sort dropdown: options for Timestamp, Filename, File Size, Dimensions. Click toggles direction (asc/desc indicator arrow).
   - Center section:
     - Grouping slider: range input from 500ms to 60000ms (0.5s to 60s). Logarithmic feel preferred (small values more sensitive). Label shows current value in human-readable format (e.g., "5s"). Debounced onChange (150ms) to avoid excessive re-grouping. Default: 5000ms (5s).
     - Slider step: use discrete steps -- 500, 1000, 2000, 3000, 5000, 10000, 15000, 30000, 60000ms
   - Right section:
     - File type filter chips: JPG, PNG, TIFF, WebP -- toggleable, multi-select. Active chips highlighted. All active by default (empty filterExtensions = show all). Clicking a chip toggles that extension.
     - Classification filter chips: Keep, Review, Delete -- toggleable, single-select (click to filter, click again to clear). Colored to match borders.
     - Search input: text field with search icon, debounced (300ms). Placeholder: "Search by filename..."
     - Thumbnail size toggle: three buttons (S/M/L) with active state highlight
   - Styling: `bg-gray-800 border-b border-gray-700 px-4 py-2 flex items-center gap-4`
   - EXIF extraction progress: if `exifProgress.completed < exifProgress.total`, show small progress text "Extracting metadata: 234/1000"

2. Create `DropZone.tsx`:
   - Full-window drop target (covers entire app)
   - Uses `onDragEnter`/`onDragOver`/`onDragLeave`/`onDrop` on a wrapper div
   - During drag-over: shows overlay with "Drop folder to open" message (semi-transparent bg-blue-900/80 overlay)
   - On drop: use `e.dataTransfer.items[i].webkitGetAsEntry()` to check if entry is directory
     - If directory: extract path from `e.dataTransfer.files[i].path` (Electron exposes .path on File objects) and call openFolder
     - If file (not directory): show brief toast/message "Drop a folder, not a file" (simple state-based message that auto-dismisses after 3s)
   - Prevent default on dragOver to allow drop
   - Track `isDragOver` state for overlay visibility

3. Create `useKeyboardNav.ts`:
   - Hook that manages keyboard navigation within the grouped grid
   - Attach keydown listener to the grid container element (passed via ref)
   - Navigation model:
     - **Left/Right arrow**: move to prev/next thumbnail within the current group. At group boundary, wrap to next/prev group.
     - **Up/Down arrow**: move to the same column position in the prev/next group. If target group has fewer images, clamp to last image.
     - **Home**: jump to first image of first group
     - **End**: jump to last image of last group
     - **Space**: cycle classification of focused image (red -> yellow -> green -> red)
   - The hook receives `groups: PhotoGroup[]` and returns `focusedImagePath: string | null`, `handleKeyDown: (e: KeyboardEvent) => void`
   - When focus changes, scroll the focused item into view (call virtualizer.scrollToIndex if needed)

4. Rewrite `App.tsx` to wire everything together:
   - Structure:
     ```
     <DropZone onFolderDrop={openFolder}>
       <div className="flex flex-col h-screen bg-gray-900 text-white">
         <Toolbar ... />
         <div className="flex-1 overflow-hidden">
           {isLoading ? <LoadingState /> :
            groups.length === 0 && folderPath ? <EmptyState /> :
            !folderPath ? <WelcomeState /> :
            <PhotoGrid ... />}
         </div>
       </div>
     </DropZone>
     ```
   - WelcomeState: initial state before any folder is opened. "Select a folder or drag one here to start". Centered, with Open Folder button.
   - LoadingState: "Scanning folder..." with progress if available
   - Listen for `window.api.onMenuOpenFolder` to handle Cmd+O from menu
   - On mount: load session, auto-open last folder if available
   - Permission error handling: if scanFolder throws, show inline error banner "Cannot access [folder] -- check permissions" (red bg banner below toolbar)
   - Large folder progress (10,000+ images): show EXIF extraction progress in toolbar (already handled by Toolbar component)
