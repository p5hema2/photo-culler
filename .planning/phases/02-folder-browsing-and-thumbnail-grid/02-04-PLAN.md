  <action>
1. Create `ExecutePanel.tsx`:
   - A modal/panel that appears when user clicks "Execute" button in toolbar
   - Shows summary of current classifications:
     - "X images marked as Keep"
     - "Y images marked as Review"
     - "Z images marked as Delete"
   - Action options (radio buttons or clear buttons):
     - **Move 'delete' images to OS trash** (default, safe): calls `window.api.trashFiles` with paths of all delete-classified images
     - **Permanently delete 'delete' images**: shows extra warning "This cannot be undone!", calls `fs.unlink` via a new IPC or uses trash with a flag. Actually -- per CONTEXT.md the user chooses "move to OS trash (recoverable) or permanent delete" each time. Implement both options as radio/button choice.
     - **Move 'keep' images to picks/ subfolder** (optional checkbox): calls `window.api.moveToPicks` with paths of all keep-classified images
   - "Execute" confirmation button -- when clicked:
     - Show a confirmation dialog: "Move Z images to Trash?" or "Permanently delete Z images?" (include count)
     - On confirm: execute the operations, show progress
     - On success: remove affected images from state, update grid immediately
     - On partial failure: show which files failed, keep them in grid
   - "Cancel" button to close without action
   - Styling: modal overlay with `bg-gray-800 rounded-lg shadow-xl p-6` card

2. Add "Execute" button to `Toolbar.tsx`:
   - Positioned at right end of toolbar
   - Shows count: "Execute (Z)" where Z is number of images classified as 'delete'
   - Disabled if no images are classified as 'delete' (no action to take)
   - Styled as a prominent action button: `bg-red-600 hover:bg-red-700` when there are deletes, `bg-gray-600 cursor-not-allowed` when disabled

3. Wire results auto-save in `usePhotoStore.ts`:
   - After `classifications` state changes, trigger the debounced save from `useResults` hook
   - Build `ResultsFile` object from current state: `{ version: 1, folderPath, updatedAt: new Date().toISOString(), images: classificationMap }`
   - On folder open: if `loadResults` returns data, merge into `classifications` state (existing results restore previous work)
   - Ensure the debounced save flushes on window unload (beforeunload event) to prevent data loss

4. Wire execute results back into state in `usePhotoStore.ts`:
   - Add `executeActions()` method that:
     - Gets all delete-classified image paths, calls trashFiles or permanent delete
     - Optionally gets all keep-classified image paths, calls moveToPicks
     - On success for each file: remove from `images` state, remove from `classifications`, clear thumbnail cache for that file
     - Save updated results file immediately (not debounced)
     - If folder is now empty after execute, show EmptyState

5. Update `App.tsx`:
   - Add ExecutePanel as a conditional modal overlay (controlled by `showExecutePanel` state)
   - Pass execute handler to Toolbar's Execute button onClick
