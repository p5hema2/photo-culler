1. Create `apps/desktop/src/renderer/src/workers/thumbnail.worker.ts`:
   - Listens for messages: `{ id: string; url: string; size: number }`
   - Fetches the image via the provided `app://` URL
   - Calls `createImageBitmap(blob)` to decode off-thread
   - Creates `OffscreenCanvas(size, size)` and draws with object-fit:cover math:
     - `scale = Math.max(size / bitmap.width, size / bitmap.height)`
     - Compute source rect to center-crop
     - `ctx.drawImage(bitmap, sx, sy, sw, sh, 0, 0, size, size)`
   - Call `bitmap.close()` to free intermediate bitmap memory
   - Convert canvas to blob: `canvas.convertToBlob({ type: 'image/jpeg', quality: 0.8 })`
   - Create final bitmap: `createImageBitmap(thumbnailBlob)`
   - Transfer back: `self.postMessage({ id, bitmap }, [bitmap])`
   - On error: `self.postMessage({ id, error: true })` -- handles TIFF or corrupt files gracefully

2. Create `apps/desktop/src/renderer/src/workers/exif.worker.ts`:
   - Import `exifr` (it works in Web Workers)
   - Listens for messages: `{ files: Array<{ path: string; url: string }> }`
   - For each file: fetch via app:// URL, get ArrayBuffer, call `exifr.parse(buffer, { pick: ['DateTimeOriginal', 'ImageWidth', 'ImageHeight'], translateValues: false })`
   - Post back each result individually: `{ path, dateTaken: timestamp | null, width: number | null, height: number | null }`
   - After all files: post `{ done: true }`
   - Catch errors per-file (don't let one bad file stop the batch), post `{ path, dateTaken: null, width: null, height: null }`

3. Create `apps/desktop/src/renderer/src/hooks/useThumbnailWorker.ts`:
   - Export `useThumbnailWorker()` hook
   - On mount: create `navigator.hardwareConcurrency` workers (or default 4) using `new Worker(new URL('../workers/thumbnail.worker.ts', import.meta.url), { type: 'module' })`
   - Store workers in a ref (do NOT recreate on render)
   - Maintain a `Map<string, ImageBitmap>` cache in a ref (NOT in React state -- ImageBitmap is not serializable)
   - Use a `thumbnailVersion` counter in state to trigger re-renders when cache updates
   - **Priority queue**: maintain a sorted array of pending requests. Accept a `visibleRange` parameter (first/last visible group indices). Items in visible range sort to front.
   - Expose: `requestThumbnail(id: string, url: string, size: number)`, `getThumbnail(id: string): ImageBitmap | 'loading' | 'error'`, `updateVisibleRange(first: number, last: number)`, `clearAll()` (for folder change -- terminates workers, creates new ones, clears cache)
   - On worker message: store bitmap in cache, increment version, dispatch next item from queue to that worker
   - On folder change (`clearAll`): terminate all workers, close all cached ImageBitmaps via `.close()`, create fresh workers

4. Create `apps/desktop/src/renderer/src/hooks/useExifExtractor.ts`:
   - Export `useExifExtractor()` hook
   - On call to `extractAll(files: Array<{ path: string; url: string }>)`: create a single EXIF worker, send all files
   - As results stream back, call a provided `onResult(path: string, dateTaken: number | null, width: number | null, height: number | null)` callback
   - When `done` message received, terminate worker
   - Expose: `extractAll(files, onResult)`, `isExtracting: boolean`, `progress: { completed: number; total: number }`

5. Create `apps/desktop/src/renderer/src/lib/results.ts`:

   - `saveResults(folderPath: string, results: ResultsFile)`: calls `window.api.saveResults(folderPath, JSON.stringify(results, null, 2))`
   - `useDebouncedSave(folderPath: string, results: ResultsFile | null, delayMs = 500)`: useEffect that debounces saves. On every results change, reset a timer. When timer fires, save. Also save on unmount (flush).
   - Mark `updatedAt` as ISO 8601 string on every save
