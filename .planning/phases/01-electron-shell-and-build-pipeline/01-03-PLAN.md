  <name>Task 1: Configure electron-builder packaging for macOS and Windows</name>
  <files>
    apps/desktop/electron-builder.yml,
    apps/desktop/build/entitlements.mac.plist,
    apps/desktop/build/icon.png
  </files>
  <action>
    **apps/desktop/electron-builder.yml:**
    Per research Pattern 4. Key points:
    - appId: "com.photo-culler.app"
    - productName: "Photo Culler"
    - directories.buildResources: "build", directories.output: "dist"
    - files: ["out/**", "package.json", "!node_modules"] -- CRITICAL: only include electron-vite output, not source or node_modules. This prevents the 300MB bloat pitfall.
    - npmRebuild: false (no native deps)
    - asar: true

    mac section:
    - category: "public.app-category.photography"
    - target: dmg with arch [x64, arm64] (Intel + Apple Silicon per DIST-01)
    - hardenedRuntime: true
    - gatekeeperAssess: false
    - entitlements and entitlementsInherit: "build/entitlements.mac.plist"
    - notarize: true (works when signing creds are present, silently skips when absent)

    win section:
    - target: nsis with arch [x64]

    nsis section:
    - oneClick: false
    - allowToChangeInstallationDirectory: true

    publish section:
    - provider: github

    **apps/desktop/build/entitlements.mac.plist:**
    Per research -- standard Electron entitlements:
    - com.apple.security.cs.allow-jit: true
    - com.apple.security.cs.allow-unsigned-executable-memory: true
    - com.apple.security.cs.allow-dyld-environment-variables: true
    - com.apple.security.files.user-selected.read-write: true

    **apps/desktop/build/icon.png:**
    Create a simple 512x512 placeholder icon. Use a minimal SVG-to-PNG or just create a 1x1 pixel PNG placeholder. electron-builder will warn about icon quality but will still build. A real icon can be added later.

    Add packaging script to apps/desktop/package.json:
    - "package:mac": "electron-builder --mac"
    - "package:win": "electron-builder --win"
    - "package": "electron-builder"

    Test local packaging: Run `pnpm build && pnpm --filter @photo-culler/desktop package:mac` (on macOS). This validates the entire pipeline: electron-vite build -> electron-builder package. Verify the output in apps/desktop/dist/ -- the .dmg should be <100MB (if >100MB, the files config is wrong and source/node_modules are leaking in).

    NOTE: Unsigned builds will show warnings but still produce working installers for development testing. Code signing activates when CI secrets are configured.

    **.github/workflows/ci.yml:**
    Per research code example. Runs on push to main and pull_request to main.
    - Matrix strategy: [macos-latest, windows-latest]
    - Steps: checkout, pnpm/action-setup@v4, setup-node@v4 (node 20, cache pnpm), pnpm install --frozen-lockfile, pnpm format:check, pnpm lint, pnpm typecheck, pnpm build
    - Add native addon detection step: `pnpm ls --depth Infinity 2>/dev/null | grep -iE "node-gyp|prebuild-install|node-pre-gyp|cmake-js"` -- fail if any found (INFRA-02 enforcement)

    **.github/workflows/release.yml:**
    Per research code example. Runs on push tags matching 'v*'.
    - Matrix: macos-latest (platform: mac), windows-latest (platform: win)
    - Steps: checkout, pnpm setup, node setup, pnpm install, pnpm build, electron-builder --${{ matrix.platform }}
    - Environment variables for code signing:
      - macOS: CSC_LINK, CSC_KEY_PASSWORD, APPLE_API_KEY, APPLE_API_KEY_ID, APPLE_API_KEY_ISSUER (from secrets)
      - Windows: WIN_CSC_LINK, WIN_CSC_KEY_PASSWORD (from secrets)
    - Upload artifacts step: actions/upload-artifact@v4 with path apps/desktop/dist/*
    - Create GitHub Release step: use softprops/action-gh-release@v2 to create a release and upload the installer artifacts

    Both workflows use pnpm/action-setup@v4 (reads packageManager from package.json automatically).

    Ensure all YAML is valid -- test locally if possible with `act` or just verify syntax.
